当然。**“只给单条命令临时加速”**的意思是：不 `export` 代理变量，不修改系统配置，不修改 `~/.bashrc`，也不改全局 Git/Pip 配置，而是在命令前面临时加一段代理环境变量。

前提是你已经在学校服务器上开好了 SOCKS5 代理：

```bash
ssh -fN -D 127.0.0.1:31880 用户名@你的加速服务器IP \
  -o ExitOnForwardFailure=yes \
  -o ServerAliveInterval=30 \
  -o ServerAliveCountMax=3
```

之后，所有临时加速命令都围绕这个代理地址：

```bash
socks5h://127.0.0.1:31880
```

这里推荐 `socks5h`，不是 `socks5`。
`h` 的意思是 hostname 也交给代理端解析，这对 GitHub、pip 下载等更稳。

------

## 1. 单条命令临时加速的基本形式

普通命令是：

```bash
pip install numpy
```

临时加速写成：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 pip install numpy
```

这表示：**只有这一次 `pip install numpy` 走代理**。

命令结束后，代理环境变量自动失效。你当前 shell 里不会留下代理设置。

你可以验证：

```bash
echo $ALL_PROXY
```

通常不会输出任何东西，因为你没有 `export`。

## 2. pip 临时加速

### 安装单个包

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install numpy
```

### 安装 requirements.txt

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install -r requirements.txt
```

### 安装 GitHub 上的 Python 包

比如：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install git+https://github.com/user/repo.git
```

这种情况很适合临时代理，因为 pip 会访问 GitHub。

### 指定 pip index 的情况

如果你还想配合官方源：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install -i https://pypi.org/simple numpy
```

或者下载大模型相关依赖：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install transformers datasets accelerate
```

## 3. GitHub clone 临时加速

推荐写法：

```bash
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    clone https://github.com/user/repo.git
```

这个写法的好处是：**只对这一次 Git 命令生效**，不会修改你的 Git 全局配置。

也可以用环境变量：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
git clone https://github.com/user/repo.git
```

不过对 Git 来说，我更推荐 `git -c`，因为它明确告诉 Git 的 HTTP/HTTPS 层使用代理。

## 4. Git pull / submodule 临时加速

### pull

```bash
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    pull
```

### submodule update

很多深度学习项目会有 submodule，比如 detectron2、一些 CUDA kernel 项目。

```bash
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    submodule update --init --recursive
```

### clone 时直接拉 submodule

```bash
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    clone --recursive https://github.com/user/repo.git
```

## 5. curl / wget 临时加速

### curl

```bash
curl --socks5-hostname 127.0.0.1:31880 https://github.com
```

下载文件：

```bash
curl --socks5-hostname 127.0.0.1:31880 -L -O https://github.com/user/repo/archive/refs/heads/main.zip
```

### wget

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
wget https://github.com/user/repo/archive/refs/heads/main.zip
```

有些版本的 `wget` 对 SOCKS 支持不如 `curl` 稳，所以下载 GitHub 文件时我更推荐 `curl`。

## 6. conda 临时加速

如果你用 conda 安装包，可以这样：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
HTTP_PROXY=socks5h://127.0.0.1:31880 \
conda install pytorch torchvision pytorch-cuda=12.1 -c pytorch -c nvidia
```

或者创建环境：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
conda create -n dl python=3.10
```

不过 conda 有时候对 SOCKS 代理支持不如 HTTP 代理稳定。如果 conda 不吃 SOCKS，可以优先用 pip，或者在加速服务器上转一层 HTTP proxy。

## 7. Hugging Face / datasets 临时加速

深度学习服务器经常需要下载模型：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
python download_model.py
```

比如 Python 脚本里有：

```python
from transformers import AutoModel

model = AutoModel.from_pretrained("bert-base-uncased")
```

你可以这样跑：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
python your_script.py
```

这样只有这一次模型下载走代理，训练本身不一定走代理。

## 8. Jupyter Notebook 里怎么做到“单次临时”

Notebook 的 `!` 命令也可以这样：

```python
!ALL_PROXY=socks5h://127.0.0.1:31880 HTTPS_PROXY=socks5h://127.0.0.1:31880 pip install transformers
```

或者 clone：

```python
!git -c http.proxy=socks5h://127.0.0.1:31880 -c https.proxy=socks5h://127.0.0.1:31880 clone https://github.com/user/repo.git
```

这比在 Notebook 里全局设置：

```python
os.environ["ALL_PROXY"] = ...
```

更干净，因为它只影响当前这一条 `!pip` 或 `!git` 命令。

## 9. 什么时候用 `ALL_PROXY`，什么时候用 `HTTPS_PROXY`

简单建议：

### 对 pip / Python / Hugging Face

用：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
命令
```

例如：

```bash
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install transformers
```

### 对 GitHub clone

优先用：

```bash
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    clone https://github.com/user/repo.git
```

### 对 curl

优先用：

```bash
curl --socks5-hostname 127.0.0.1:31880 URL
```

## 10. 不推荐这样做

不推荐你一上来就：

```bash
export ALL_PROXY=socks5h://127.0.0.1:31880
export HTTPS_PROXY=socks5h://127.0.0.1:31880
```

因为这会影响当前终端后续所有命令。你可能会忘记关掉，导致一些校内资源、内网服务、Jupyter 扩展、conda 源访问变慢或异常。

更不推荐写进：

```bash
~/.bashrc
~/.zshrc
~/.profile
```

因为那样每次登录都会自动走代理，不符合你“临时加速”的需求。

## 11. 一个适合你的常用模板

你可以在学校服务器上这样用：

```bash
# 先开 SOCKS5 代理
ssh -fN -D 127.0.0.1:31880 用户名@你的加速服务器IP \
  -o ExitOnForwardFailure=yes \
  -o ServerAliveInterval=30 \
  -o ServerAliveCountMax=3

# 测试 GitHub
curl --socks5-hostname 127.0.0.1:31880 https://github.com

# 临时 clone
git -c http.proxy=socks5h://127.0.0.1:31880 \
    -c https.proxy=socks5h://127.0.0.1:31880 \
    clone https://github.com/user/repo.git

# 临时 pip
ALL_PROXY=socks5h://127.0.0.1:31880 \
HTTPS_PROXY=socks5h://127.0.0.1:31880 \
pip install -r requirements.txt

# 用完关掉 SSH 代理
pkill -f "ssh -fN -D 127.0.0.1:31880"
```

## 12. 可以做一个临时函数，方便但不污染系统

你可以在当前 shell 里定义一个函数：

```bash
proxycmd() {
  ALL_PROXY=socks5h://127.0.0.1:31880 \
  HTTPS_PROXY=socks5h://127.0.0.1:31880 \
  HTTP_PROXY=socks5h://127.0.0.1:31880 \
  "$@"
}
```

然后这样用：

```bash
proxycmd pip install -r requirements.txt
proxycmd python download_model.py
proxycmd curl -L https://github.com
```

这个函数只存在于当前终端，关闭终端就没了。它比全局 `export` 更安全，也比每次手打环境变量方便。

对于 Git，可以再做一个函数：

```bash
gitproxy() {
  git -c http.proxy=socks5h://127.0.0.1:31880 \
      -c https.proxy=socks5h://127.0.0.1:31880 \
      "$@"
}
```

然后：

```bash
gitproxy clone https://github.com/user/repo.git
gitproxy pull
gitproxy submodule update --init --recursive
```

这套方案很适合你的场景：**学校服务器保持公开服务不变，只有你当前执行的 GitHub / pip / 模型下载命令临时走加速服务器。**
