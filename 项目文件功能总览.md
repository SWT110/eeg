# 项目文件功能总览

本文档用于替代主目录下多份分散说明文档，重点说明当前项目中每个主要文件、目录和产物的作用。后续清理旧文档时，建议保留本文档作为唯一入口。

## 项目定位

本项目围绕 EEG 活动三分类流程组织，当前主线是：

```text
原始 EEG / 日志数据
-> 数据整理、切段、归一化
-> 构建窗口级全局活动数据集
-> EEG-Conformer LOSO 三分类训练
-> 汇总评估结果
-> 训练最终部署模型
-> 对新 xlsx 信号做预测或通过 API 演示
```

代码、测试、配置和说明文档适合通过 GitHub 同步；原始数据、中间产物、训练输出、API 缓存和视频统一放到根目录 `local_artifacts/`，不进入 Git。

## 路径配置文件

### `eeg_project_paths.py`

这是全项目共享的路径中心。训练脚本和数据处理脚本不要各自硬编码数据目录，而是从这里取默认路径。

| 名称 | 默认指向 | 作用 |
| --- | --- | --- |
| `LOCAL_ARTIFACTS_ROOT` | `local_artifacts/` | 所有本地大文件和输出的根目录。 |
| `DATA_TO_LIST_DATA_DIR` | `local_artifacts/data_to_list/data/` | 原始 EDF、日志等数据输入目录。 |
| `LIST_DIR` | `local_artifacts/data_to_list/list/` | 日志或 EDF 转换后的列表化 Excel 目录。 |
| `LIST_CUT_DIR` | `local_artifacts/data_to_list/list_cut_fixed_duration/` | 按固定时长切分后的数据目录。 |
| `LIST_NORMALIZED_DIR` | `local_artifacts/data_to_list/list_normalization_fixed_duration/` | 归一化后的 EEG Excel 目录，也是训练数据构建的常用输入。 |
| `NPY_DATASET_DIR` | `local_artifacts/data_to_list/npy_dataset/` | subject-level `.npy` 数据集输出目录。 |
| `GLOBAL_ACTIVITY_DATASET_DIR` | `local_artifacts/data_to_list/global_activity_dataset/` | LOSO 和最终模型使用的全局活动数据集目录。 |
| `WINDOW_STRIDE_CONFIG` | `eeg-data-processing/data_to_list/window_stride_configs.json` | 多套窗口长度和步长配置。 |
| `ACTIVITY_LOSO_OUTPUT_DIR` | `local_artifacts/outputs/activity_loso/` | LOSO 单折和批量训练输出目录。 |
| `ACTIVITY_FINAL_OUTPUT_DIR` | `local_artifacts/outputs/activity_final/` | 最终部署模型输出目录。 |
| `ACTIVITY_API_CACHE_DIR` | `local_artifacts/outputs/activity_api_cache/` | API 预测缓存目录。 |
| `SORT2_TIME_DATA_DIR` | `local_artifacts/data_to_sorted/sort2/time_data/` | sort2 反馈时间文件目录。 |
| `VIDEOS_DIR` | `local_artifacts/videos/` | 本地视频目录。 |

训练服务器如果把大文件放在其他磁盘，可以设置：

```bash
export EEG_LOCAL_ARTIFACTS_ROOT=/path/to/local_artifacts
```

也可以按目录单独覆盖，例如 `EEG_GLOBAL_ACTIVITY_DATASET_DIR`、`EEG_ACTIVITY_LOSO_OUTPUT_DIR`、`EEG_ACTIVITY_FINAL_OUTPUT_DIR`。

## 训练数据产物含义

### 全局活动数据集

全局活动数据集是训练主线的核心输入，通常位于：

```text
local_artifacts/data_to_list/global_activity_dataset/
├── X.npy
├── y.npy
├── subject_ids.npy
├── record_ids.npy
├── window_indices.npy
└── metadata.json
```

| 文件 | 功能 |
| --- | --- |
| `X.npy` | 所有 EEG 窗口数据，形状通常为 `(N, C, T)`。 |
| `y.npy` | 每个窗口对应的活动类别标签，当前是三分类。 |
| `subject_ids.npy` | 每个窗口来自哪个受试者，用于 LOSO 划分。 |
| `record_ids.npy` | 每个窗口来自哪一段原始活动记录。 |
| `window_indices.npy` | 每个窗口在原始连续记录中的顺序。 |
| `metadata.json` | 标签映射、窗口秒数、步长、样本统计等元信息。 |

### LOSO 输出

```text
local_artifacts/outputs/activity_loso/
├── fold_subject_1/
├── fold_subject_2/
├── ...
├── summary.json
└── summary.csv
```

每个 `fold_subject_<id>/` 一般包含：

| 文件 | 功能 |
| --- | --- |
| `best_model.pt` | 当前 fold 上保存的最佳模型 checkpoint。 |
| `metrics.json` | 当前 fold 的训练、验证、测试指标。 |
| `test_predictions.npz` | 测试集真实标签和预测标签，供汇总使用。 |
| `epoch_history.csv` | 每个 epoch 的损失和准确率，适合画训练曲线。 |
| `epoch_history.json` | 与 CSV 对应的结构化训练历史。 |
| `train.log` | 当前 fold 的完整训练日志。 |

### 最终部署模型输出

```text
local_artifacts/outputs/activity_final/
├── final_model.pt
├── train_config.json
├── normalization_stats.json
├── label_map.json
└── val_metrics.json
```

| 文件 | 功能 |
| --- | --- |
| `final_model.pt` | 实际预测新 EEG 信号时加载的最终模型。 |
| `train_config.json` | 保存训练参数和模型结构参数。 |
| `normalization_stats.json` | 保存训练集均值和标准差，推理时必须使用同一套标准化参数。 |
| `label_map.json` | 保存类别名和数字标签的映射。 |
| `val_metrics.json` | 最终模型训练时的验证指标。 |

## `EEG-Conformer/` 文件

`EEG-Conformer/` 是当前活动三分类模型子项目。它不是上游仓库的原样镜像，而是保留 EEG-Conformer 模型思路后，针对当前数据流程重写的训练、评估、推理和 API 入口。

| 文件 | 功能 |
| --- | --- |
| `EEG-Conformer/README.md` | 子项目说明，简述保留脚本、数据输入和常用命令。内容已被本文吸收，后续可只保留本文。 |
| `EEG-Conformer/LICENSE` | 上游 EEG-Conformer 相关许可文件。 |
| `EEG-Conformer/.gitignore` | 子目录级忽略配置，保护子项目内环境和输出。 |
| `EEG-Conformer/train_activity_loso.py` | 单个 LOSO fold 的正式三分类训练入口；内含 EEG-Conformer 模型定义、数据加载、训练、评估和 checkpoint 保存逻辑。 |
| `EEG-Conformer/train_activity_loso_batch.py` | 对一套全局数据集批量跑多个或全部 LOSO fold，复用单 fold 训练脚本。 |
| `EEG-Conformer/train_activity_loso_generated_batch.py` | 遍历多套 `window_<...>_stride_<...>` 数据集，批量运行 LOSO 并自动汇总，适合比较不同窗口长度和步长。 |

| `EEG-Conformer/train_activity_final.py` | 用全局数据集训练最终部署模型，输出 `final_model.pt`、训练配置、归一化参数、标签映射和验证指标。 |
| `EEG-Conformer/predict_activity_signal.py` | 加载最终模型，对单个 EEG `.xlsx` 文件切窗预测，汇总窗口概率后输出整段活动类别。 |
| `EEG-Conformer/app_activity_api.py` | FastAPI 服务入口，提供模型选择、文件上传和活动预测接口，并使用 API cache 保存临时结果。 |
| `EEG-Conformer/index.html` | 对应 FastAPI 的简单网页界面，用于浏览器上传 EEG 文件并查看预测结果。 |
| `EEG-Conformer/setup_env_jupyter.py` | 面向 CUDA 服务器的项目内 Conda 环境和 Jupyter kernel 创建脚本，不是 Mac ARM 环境主入口。 |

## 主要运行流程

### 1. 从归一化 Excel 构建全局数据集

```bash
python eeg-data-processing/data_to_list/build_activity_global_index.py \
  --input-root local_artifacts/data_to_list/list_normalization_fixed_duration \
  --output-root local_artifacts/data_to_list/global_activity_dataset \
  --window-seconds 5 \
  --stride-seconds 5
```

### 2. 批量生成多套 window/stride 数据集

```bash
python eeg-data-processing/data_to_list/generate_datasets.py \
  --config eeg-data-processing/data_to_list/window_stride_configs.json \
  --input-root local_artifacts/data_to_list/list_normalization_fixed_duration \
  --output-base local_artifacts/data_to_list/global_activity_dataset
```

### 3. 单 fold LOSO 调试

## GitHub 同步边界

适合进入 GitHub：

- Python 脚本
- 测试文件
- README / Markdown 文档
- 配置文件，例如 `window_stride_configs.json`
- 依赖清单，例如 `requirements-mac-test-train.txt`

不适合进入 GitHub：

- `local_artifacts/`
- `.conda-envs/`、`.conda-pkgs/`、`.venv*/`
- `__pycache__/`、`.pytest_cache/`、`.ipynb_checkpoints/`
- EDF、xlsx、npy、pt、视频、训练日志和 API 上传缓存

同步建议：

```bash
zip -r local_artifacts.zip local_artifacts
```

服务器解压后保持目录名为 `local_artifacts/`，或设置 `EEG_LOCAL_ARTIFACTS_ROOT` 指向解压目录。
