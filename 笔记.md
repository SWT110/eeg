# 三分类训练命令笔记

当前不需要再手动改 `train_activity_loso.py` 里的 loss。类别权重已经支持命令行参数：

```bash
--class-weights 3,3,1
```

现在推荐默认跑双分支：

```bash
--input-domain time_fft
```

含义：

```text
时域 EEG 分支 + FFT 频域分支 -> 特征拼接 -> 三分类
```

## 4. 参数说明：train_activity_loso_generated_batch.py

这个脚本用于按 `window_stride_configs.json` 批量跑多套窗口配置，并对每套数据集跑完整 LOSO。

### `--input-domain`

控制模型输入形式。

| 参数值 | 含义 | 说明 |
| --- | --- | --- |
| `time` | 只用时域 EEG | 原始 EEG 窗口输入，单分支模型。 |
| `fft` | 只用频域 | 对时间轴做 rFFT，取 `log(1 + power)`，单分支模型。 |
| `time_fft` | 时域 + FFT 双分支 | 推荐主实验；两个分支独立提特征后拼接分类。 |

注意：`time_fft` 的 FFT 分支要求 FFT 后长度能通过 EEG-Conformer stem。当前 `window_3_stride_3` 可以跑；太短窗口，比如 1 秒，可能因为 FFT 后长度不足而报错。

---

### `--epochs`

训练轮数。

| 推荐值 | 用途 |
| --- | --- |
| `1` | smoke test，只验证代码能跑通。 |
| `50` | 快速观察趋势。 |
| `100` | 中等实验。 |
| `200` | 当前推荐正式实验。 |

例如：

```bash
--epochs 200
```

---

### `--device`

控制训练设备。

| 参数值 | 使用场景 |
| --- | --- |
| `cpu` | 最稳，但很慢，适合 smoke test。 |
| `mps` | Apple Silicon Mac 使用。 |
| `cuda:0` | 服务器第一张 NVIDIA GPU。 |
| `cuda:1` | 服务器第二张 NVIDIA GPU。 |

例如：

```bash
--device mps
--device cuda:0
```

---

### `--batch-size`

每个 batch 的窗口数量。

| 推荐值 | 使用场景 |
| --- | --- |
| `8` | Mac MPS 或显存紧张。 |
| `16` | Mac MPS 常用。 |
| `32` | CUDA 显存一般时。 |
| `72` | 代码默认值，CUDA 显存够时可用。 |

如果报显存/内存不足，优先降低这个参数。

---

### `--class-weights`

类别权重，用于处理类别不平衡。

当前数据里活动 3 样本明显更多，所以推荐：

```bash
--class-weights 3,3,1
```

含义：

```text
活动 1 权重 3
活动 2 权重 3
活动 3 权重 1
```

可选写法：

| 参数 | 含义 |
| --- | --- |
| 不写 | 不使用类别权重。 |
| `--class-weights 1,1,1` | 等价于基本不加权。 |
| `--class-weights 3,3,1` | 当前推荐，增强活动 1/2。 |
| `--class-weights 4,4,1` | 更强地增强活动 1/2，可能提高召回，也可能降低整体准确率。 |

---

### `--output-base`

控制训练结果保存在哪里。

推荐不同实验放不同目录，避免混在一起：

```bash
--output-base local_artifacts/outputs/activity_loso_time
--output-base local_artifacts/outputs/activity_loso_fft
--output-base local_artifacts/outputs/activity_loso_time_fft
```

如果改了重要参数，比如：

```text
input-domain
class-weights
epochs
window/stride
```

建议换一个新的 `output-base`，否则可能因为已有结果被跳过，或者新旧结果混在一起。

---

### `--lr`

学习率，默认：

```bash
--lr 0.0002
```

一般先不动。可尝试：

| 参数 | 说明 |
| --- | --- |
| `--lr 0.0002` | 默认推荐。 |
| `--lr 0.0001` | 更保守，训练更慢但可能更稳。 |
| `--lr 0.0005` | 更激进，可能更快，也可能不稳定。 |

---

### `--config`

窗口/步长配置文件，默认：

```bash
--config eeg-data-processing/data_to_list/window_stride_configs.json
```

这个文件决定要跑哪些数据集，例如：

```json
[
  {"window_seconds": 3.0, "stride_seconds": 3.0}
]
```

注意：训练脚本只读取已经生成好的数据集。如果改了 config，需要先生成对应数据集。

---

### `--dataset-base`

全局数据集根目录，默认：

```bash
--dataset-base local_artifacts/data_to_list/global_activity_dataset
```

脚本会在里面找：

```text
window_3_stride_3/
window_5_stride_5/
...
```

一般不用改，除非服务器数据放在其他位置。

---

### `--skip-existing` / `--no-skip-existing`

默认是：

```bash
--skip-existing
```

也就是已有完整结果会跳过。

如果想强制重跑：

```bash
--no-skip-existing
```

但更推荐换新的 `--output-base`，避免覆盖或混淆旧结果。

---

### `--seed`

随机种子，默认：

```bash
--seed 42
```

用于控制随机初始化和训练可复现性。一般不动；如果想做多随机种子实验，可以尝试：

```bash
--seed 1
--seed 2
--seed 3
```

并配不同的 `output-base`。
